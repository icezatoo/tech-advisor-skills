# /tech-advisor:explain

Run the **tech-advisor** skill in `explain` mode. Canonical logic lives in `skills/tech-advisor/SKILL.md`; subcommand spec in `skills/tech-advisor/commands/explain.md` — load and follow both.

Explain a single tool, pattern, or decision like a tech lead walking a teammate through the "why": what it is, when to use it, when NOT to, trade-offs, how it fits a modern stack, and risks. Be honest about when it's overkill.
